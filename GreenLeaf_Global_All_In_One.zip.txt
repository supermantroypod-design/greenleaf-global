# GreenLeaf Global — replit.md

## Overview

GreenLeaf Global is a cannabis dispensary discovery and pickup ordering web app. Users can browse local dispensaries, view their product menus (with strain info, THC/CBD levels, pricing), add items to a cart, and place pickup orders. The app includes user authentication (register/login), an interactive map view of dispensaries, and an order history in user profiles.

The project is a full-stack TypeScript monorepo with:
- A **React SPA** (client) served via Vite
- An **Express REST API** (server)
- A **PostgreSQL database** accessed via Drizzle ORM
- A **shared** directory for schema definitions and typed API route contracts

## System Architecture

### Monorepo Layout

```
/
├── client/        # React frontend (Vite)
│   └── src/
│       ├── components/   # UI components (shadcn/ui + custom)
│       ├── hooks/        # React Query hooks for all data fetching
│       ├── lib/          # API client, query client, utilities
│       ├── pages/        # Route-level page components
│       └── store/        # Zustand cart store
├── server/        # Express backend
│   ├── db.ts      # Drizzle + pg Pool setup
│   ├── routes.ts  # All API route handlers
│   ├── storage.ts # Database access layer (IStorage interface)
│   ├── vite.ts    # Dev-mode Vite middleware integration
│   └── static.ts  # Production static file serving
├── shared/        # Shared between client and server
│   ├── schema.ts  # Drizzle table definitions + Zod insert schemas
│   └── routes.ts  # Typed API route contracts (path, method, input/output schemas)
└── script/build.ts # Custom esbuild + viteBuild production script
```

### Frontend Architecture

- **Framework**: React 18 with TypeScript, bundled by Vite
- **Routing**: `wouter` (lightweight client-side routing)
- **State management**:
  - **Server state**: TanStack React Query (v5) for all API calls, caching, and invalidation
  - **Client/local state**: Zustand (`use-cart.ts`) with `persist` middleware for the shopping cart (stored in `localStorage`)
- **UI components**: shadcn/ui (New York style) built on Radix UI primitives + Tailwind CSS
- **Animations**: Framer Motion for page transitions (`AnimatePresence` + `PageTransition` wrapper) and micro-animations in the mobile nav
- **Forms**: React Hook Form + Zod validation (`@hookform/resolvers`)
- **Map**: Leaflet (loaded dynamically via `import("leaflet")`) inside `DispensaryMap.tsx`
- **Fonts**: DM Sans (body) and Outfit (display), loaded from Google Fonts

**Page routes:**
| Path | Page |
|------|------|
| `/` | Home (hero + featured dispensaries) |
| `/discover` | Dispensary list/map search |
| `/dispensary/:id` | Dispensary detail + product catalog |
| `/cart` | Cart review |
| `/checkout` | Order placement |
| `/profile` | User profile + order history |
| `/auth` | Login / Register |

### Backend Architecture

- **Framework**: Express.js on Node.js, using ESM modules
- **Entry point**: `server/index.ts` creates an HTTP server and mounts Express; in dev mode it integrates Vite as middleware via `server/vite.ts`; in production it serves the built static files via `server/static.ts`
- **Route registration**: All routes live in `server/routes.ts`. A `seedDatabase()` call runs on startup to populate the DB if empty
- **Storage layer**: `server/storage.ts` defines an `IStorage` interface and a `DatabaseStorage` class that implements it using Drizzle queries. This pattern makes it easy to swap implementations
- **Authentication**: JWT-based. Tokens are signed with `jsonwebtoken` using a secret from `SESSION_SECRET` env var. The `authenticateToken` middleware validates `Authorization: Bearer <token>` headers on protected routes. Tokens are stored in `localStorage` on the client side under the key `gl_token`

### Shared API Contracts (`shared/routes.ts`)

All API routes are defined as typed objects including `method`, `path`, Zod `input` schema, and typed `responses`. Both the server (for validation) and client hooks (for parsing responses) consume these directly, ensuring end-to-end type safety without a separate code generation step.

### Database Schema (`shared/schema.ts`)

All tables use Drizzle ORM with PostgreSQL dialect:

| Table | Key Columns |
|-------|-------------|
| `users` | id, email (unique), password, name, createdAt |
| `dispensaries` | id, name, address, city, zipCode, rating, imageUrl, latitude, longitude |
| `categories` | id, name |
| `products` | id, dispensaryId, categoryId, name, strainType, thcLevel, cbdLevel, price, imageUrl |
| `orders` | id, userId, dispensaryId, status, totalAmount |
| `orderItems` | id, orderId, productId, quantity, priceAtTime |

Drizzle-Zod (`drizzle-zod`) generates insert schemas from table definitions.

### Build System

- **Dev**: `tsx server/index.ts` — runs the server directly with TypeScript, integrating Vite dev server as Express middleware
- **Production build**: `script/build.ts` runs Vite for the client, then esbuild for the server, bundling a curated allowlist of server dependencies to reduce cold start time. Output goes to `dist/`
- **DB migrations**: `drizzle-kit push` (schema push, not migration files)

### Styling

- Tailwind CSS with a fully custom dark theme defined via CSS variables in `client/src/index.css`
- Theme uses a dark green base (`--background: 140 30% 8%`) with an emerald green primary (`--primary: 142 76% 52%`)
- Custom `glass-panel` utility used on the header and mobile nav for frosted-glass effects

## External Dependencies

### Runtime Services
- **PostgreSQL**: Required. Connection via `DATABASE_URL` environment variable. Used for all persistent data
- **Google Fonts**: DM Sans and Outfit fonts loaded in `client/index.html` and `index.css`

### Key NPM Dependencies

| Package | Purpose |
|---------|---------|
| `drizzle-orm` + `pg` | Database ORM + PostgreSQL driver |
| `drizzle-zod` | Auto-generates Zod schemas from Drizzle tables |
| `express` | HTTP server / API |
| `jsonwebtoken` | JWT creation and verification |
| `@tanstack/react-query` | Server state management + data fetching |
| `zustand` | Client-side cart state with localStorage persistence |
| `wouter` | Lightweight React router |
| `framer-motion` | Page transitions and animations |
| `leaflet` | Interactive dispensary map |
| `react-hook-form` + `@hookform/resolvers` | Form state and Zod validation |
| `shadcn/ui` components | All UI primitives (built on Radix UI) |
| `tailwind-merge` + `clsx` | Conditional class name utilities |
| `date-fns` | Date formatting in order history |
| `zod` | Runtime validation for API inputs and outputs |

### Environment Variables Required
- `DATABASE_URL` — PostgreSQL connection string
- `SESSION_SECRET` — JWT signing secret (falls back to a hardcoded dev value if not set)

## Export / Local Setup

The project is fully export-ready. Key files for running outside Replit:

| File | Purpose |
|------|---------|
| `setup.sh` | One-command local setup: installs deps, creates DB, applies schema |
| `.env.example` | Environment variable template — copy to `.env` and fill in |
| `.gitignore` | Excludes `.env`, `node_modules`, `dist`, editor files |
| `database/schema.sql` | Portable plain-SQL schema (matches `shared/schema.ts`) |
| `database/seed.sql` | Portable plain-SQL demo data (21 products, 3 dispensaries, 1 user) |
| `vite.config.ts` | Replit-specific plugins guarded by `REPL_ID` check — safe locally |

### Local quickstart
```bash
git clone <repo>
cd greenleaf-global
./setup.sh           # checks prereqs, installs, creates DB, pushes schema
npm run dev          # starts app on http://localhost:5000
```
